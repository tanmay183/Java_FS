## üåê **1. What is TypeScript?**

### ‚úÖ Simple Explanation:

**TypeScript** is a **superset of JavaScript** ‚Äî meaning it includes everything JavaScript has, **plus features like:**

* **Static typing**
* **Interfaces & classes**
* **Compile-time checks**
* **Modern JS support (ES6+ features)**

üí° Angular is built using TypeScript for **safety**, **structure**, and **developer tooling**.

---

## üõ†Ô∏è **2. Superset of JavaScript with Static Typing**

### ‚úÖ What is Static Typing?

You declare variable **types** in advance (like Java or C#).

```ts
let age: number = 25;
let name: string = "Tanmay";
```

‚úÖ If you try `age = "twenty"` ‚Üí ‚ùå Error at compile time.


---

## üß™ **4. `tsc` Compiler**

### ‚úÖ Use:

`tsc` is the TypeScript **compiler** that converts `.ts` files into `.js`.

```bash
tsc main.ts      # Compiles to main.js
```

‚úÖ You can then run `main.js` with Node.js or a browser.

---

## üì¶ **5. `tsconfig.json` File**

### ‚úÖ Purpose:

Tells the compiler **how to compile** your TS code (like a config file).

### üìå Create it:

```bash
tsc --init
```

### üìå Sample:

```json
{
  "compilerOptions": {
    "target": "ES6",
    "strict": true,
    "outDir": "./dist",
    "module": "commonjs"
  },
  "include": ["src/**/*"]
}
```

‚úÖ Now just run:

```bash
tsc       # Compiles entire project
```

## üî§ **6. Basic Types in TypeScript**

---

### ‚úÖ `number`

```ts
let age: number = 30;
```

---

### ‚úÖ `string`

```ts
let name: string = "Tanmay";
```

---

### ‚úÖ `boolean`

```ts
let isActive: boolean = true;
```

---

### ‚úÖ `any`

Allows **any type** (not type safe)

```ts
let value: any = "hello";
value = 123; // Allowed
```

---

### ‚úÖ `unknown`

Like `any`, but **forces you to check** before using.

```ts
let data: unknown = "test";

if (typeof data === 'string') {
  console.log(data.toUpperCase());
}
```

‚úÖ More secure than `any`.

---

### ‚úÖ `void`

Used when a function **returns nothing**

```ts
function greet(): void {
  console.log("Hello");
}
```

---

### ‚úÖ `never`

Used when a function **never finishes** (e.g., throws error)

```ts
function error(): never {
  throw new Error("Something went wrong");
}
```

---

### ‚úÖ `null` and `undefined`

```ts
let empty: null = null;
let notSet: undefined = undefined;
```

> In strict mode, they are not automatically assignable to other types.

---

## üß∫ **7. Arrays and Tuples**

---

### ‚úÖ Arrays

```ts
let numbers: number[] = [1, 2, 3];
let fruits: Array<string> = ["apple", "banana"];
```

---

### ‚úÖ Tuples

Fixed-length, fixed-type arrays.

```ts
let person: [string, number] = ["Tanmay", 25];
```

### ‚ùå Invalid:

```ts
let person: [string, number] = [25, "Tanmay"]; // Error!
```
